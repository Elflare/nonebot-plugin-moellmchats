# 完整指令表

## 说明

- **权限**：`无` = 普通群友可用；`超级管理员` = 仅 `.env` 中 `SUPERUSERS` 配置的 QQ 号可用
- **范围**：群聊 / 私聊 / 私聊+群聊
- 指令中 `<参数>` 为必填，`[参数]` 为选填
- 同一行列出的多个指令名完全等价（均可触发相同功能）

## 工具与插件管理（超级管理员）

| 指令                                 | 范围      | 参数     | 说明                                                                                      |
| ------------------------------------ | --------- | -------- | ----------------------------------------------------------------------------------------- |
| `刷新工具` / `重载工具` / `刷新插件` | 私聊/群聊 | 无       | 热重载 NoneBot 插件描述、`custom_tools/` 自定义函数、`mcp_servers.toml` 中启用的 MCP 工具 |
| `插件黑名单` / `查看插件黑名单`      | 私聊/群聊 | 无       | 查看禁止 LLM 调用的插件/函数/MCP 工具列表                                                 |
| `添加插件黑名单 <工具标识>`          | 私聊/群聊 | 工具标识 | 禁止 LLM 调用指定 NoneBot 插件、自定义函数或 MCP 工具                                     |
| `移除插件黑名单 <工具标识>`          | 私聊/群聊 | 工具标识 | 从黑名单移除指定工具                                                                      |
| `添加常驻插件 <标识>`                | 私聊/群聊 | 工具标识 | 将 NoneBot 插件、自定义函数或 MCP 工具设为常驻                                            |
| `移除常驻插件 <标识>`                | 私聊/群聊 | 工具标识 | 从常驻列表移除                                                                            |
| `移除常驻插件 <标识>`                | 私聊/群聊 | 插件包名 | 从常驻列表移除                                                                            |
| `添加常驻函数 <标识>`                | 私聊/群聊 | 函数名   | 将 custom_tools 中的函数设为常驻                                                          |
| `移除常驻函数 <标识>`                | 私聊/群聊 | 函数名   | 从常驻列表移除                                                                            |

### 工具标识说明

工具标识可以是：

- NoneBot 插件包名：如 `nonebot_plugin_xxx`
- 自定义函数名：如 `extract_webpage`
- MCP 工具名：如 `mcp__danbooru_searcher__search_tags`
- MCP 服务级黑名单：如 `mcp__danbooru_searcher`
- MCP 通配黑名单：如 `mcp__danbooru_searcher__*`

示例：

```text
添加插件黑名单 mcp__filesystem
添加插件黑名单 mcp__filesystem__read_file
添加插件黑名单 mcp__filesystem__*
移除插件黑名单 mcp__filesystem
添加常驻插件 mcp__danbooru_searcher__search_tags
```

## 运行中请求管理（超级管理员）

| 指令                                                       | 范围      | 参数          | 说明                                                       |
| ---------------------------------------------------------- | --------- | ------------- | ---------------------------------------------------------- |
| `查看请求` / `查看当前请求` / `当前请求` / `查看正在请求`   | 私聊/群聊 | 无            | 查看当前正在处理的 LLM 请求、来源、用户、运行时长与消息预览 |
| `停止请求` / `终止请求` / `取消请求` / `stop请求` `[参数]` | 私聊/群聊 | 编号或 `all`  | 停止指定或全部正在处理的 LLM 请求；不带参数且仅有一个请求时会直接停止 |

**停止请求参数：**

| 格式 | 示例          | 含义                     |
| ---- | ------------- | ------------------------ |
| 省略 | `停止请求`    | 仅有一个运行中请求时停止它；多个请求时提示先查看编号 |
| 编号 | `停止请求 1`  | 停止编号为 1 的请求      |
| 全部 | `停止请求 all` | 停止所有正在处理的请求   |

被停止的请求会清理对应用户的 CD 状态。若请求已经进入流式输出或工具执行阶段，已经发出的消息或已经执行的工具副作用不会被撤回。

## Token 消耗查询（超级管理员）

| 指令                                            | 范围      | 参数               | 说明                    |
| ----------------------------------------------- | --------- | ------------------ | ----------------------- |
| `查看消耗` / `查询token` / `token消耗` `[参数]` | 私聊/群聊 | 数量或范围（选填） | 查看 API Token 消耗记录，新记录会显示本轮请求累计耗时 |

**参数格式：**

| 格式 | 示例             | 含义                          |
| ---- | ---------------- | ----------------------------- |
| 省略 | `查看消耗`       | 最近 5 条                     |
| 数字 | `查看消耗 10`    | 最近 10 条                    |
| 区间 | `查看消耗 10-15` | 第 10 到第 15 条（1-indexed） |
| 负数 | `查看消耗 -50`   | 最早的 50 条                  |

新版本产生的消耗记录会在模型名后显示耗时，例如：

```text
[1] 05-22 16:58:00 | gpt-4o-mini | 耗时 12.3s
 ├ 提示词: 1234 (其中命中缓存: 500)
 ├ 生成:   233
 └ 总计:   1467
```

这里的耗时为本轮 LLM 请求从登记开始到该次 API usage 被记录时的累计耗时；旧记录没有耗时字段时会保持原样。
